# DESIGN.md — るうまな（RUUMANA）デザイン仕様書

> **目的:** 本ドキュメントは「るうまな」のブランド・UI・UX を、コードや派生プロダクト（公式サイト・販促物・パートナー資料等）の生成に再利用できる形に抽出した、ビジュアル＆デザインの SoT。
>
> **適用範囲:** ロゴ・色・タイポグラフィ・トーン＆マナー・情報設計・コンポーネント・モーション。技術仕様や業務要件は `docs/spec/` を、企画・市場情報は `docs/企画/`・`docs/事業概要/` を参照すること。

## 1. ブランドアイデンティティ

### 1.1 サービス名と由来

| 項目 | 内容 |
|------|------|
| サービス名 | るうまな（RUUMANA） |
| 由来 | 「ルール（Rule）」と「マナー（Manner）」の造語 |
| タグライン（短） | 異文化の摩擦を、理解へ。 |
| タグライン（コンセプト） | 知りたいマナーや解決策に、1タップでアクセス。 |
| ミッション | 異なる文化を持つ人同士が安心して交流できる社会を実現する |

### 1.2 ブランドパーソナリティ（ボイス＆トーン）

| 軸 | 取る | 取らない |
|----|------|---------|
| 姿勢 | 温かい・親しみやすい | 事務的・上から目線 |
| 態度 | 否定しない・尊重する | 正誤・優劣で裁く |
| 言い回し | 平易・短文・楽しげ | 専門用語・難語・命令調 |
| 視点 | 一人ひとりの「個」 | 国籍・属性のラベル |
| 学び方 | 楽しく・好奇心ドリブン | 堅苦しい・義務的 |

### 1.3 5つの理念（ブランドの根）

1. **否定しない** — 文化の違いを優劣や正誤で判断しない
2. **争わない** — 違いで対立せず橋渡しになる
3. **個を尊重する** — 国籍や属性でなく一人の人として向き合う
4. **共生** — 相互理解と尊重に基づく共生社会
5. **楽しく学ぶ** — 堅苦しくない、温かい文化体験

> 派生コンテンツ（コピー・ビジュアル）でも、この5つに反する表現（例：「日本では〇〇は禁止です」「マナー違反は恥」など断罪的トーン）を採用しない。

## 2. ロゴと象徴ビジュアル

### 2.1 ロゴ

- 配置先: `/public/images/logo.png`, `/public/icons/logo.png`
- 使用シーン: ヘッダー、ローディング、ランディングのヒーロー
- 周囲には「リング（円形）」のフレームをまとわせる場合がある（ランディングの `landing-logo-ring`）
- 単独使用時は背景にコントラストを十分に確保し、上下左右にロゴ高さの 1/2 以上の余白を取る

### 2.2 モチーフ：旅と空

| モチーフ | 用途 |
|---------|------|
| 空（DynamicSky / SkyAnimation） | ヒーロー・トップ背景。時間帯で色が変化する動的グラデーション |
| 荷物タグ（LuggageTag, `rmp_luggage_*.png`） | 旅・出発・ワクワク感の象徴 |
| 国旗（SkyAnimation 内） | 多文化・多国籍を象徴 |
| 4色のアイコンタイル | 「しる・みる・きく・いく」の4つの柱 |

これらは「旅 × 出会い × 異文化交流」というブランドの世界観を補強する。固いコーポレート的ビジュアルは避ける。

## 3. カラーパレット

### 3.1 機能色（4つの柱と緊急）

| 名称 | HEX | Tailwind | 用途 |
|------|-----|---------|------|
| しるまな Green | `#85C441` | `ru-shiru` | しるまな（知る）関連の全要素 |
| みるまな Orange | `#F26522` | `ru-miru` | みるまな（見る）関連 |
| きくまな Pink | `#E4007F` | `ru-kiku` | きくまな（聞く）関連・主要 CTA |
| いくまな Blue | `#0099DD` | `ru-iku` | いくまな（行く）関連 |
| おたすけ Red | `#E60012` | `ru-trouble` | 緊急・エラー・警告 |

### 3.2 ベース色・テキスト色

| 名称 | HEX | 用途 |
|------|-----|------|
| Beige Background | `#F9F5E7` (`ru-bg`) | 全体背景（クリームベージュ） |
| White Surface | `#FFFFFF` | カード・モーダル・セクション分け |
| Primary Purple | `#4B3E8E` | 見出し・チャット・主要アクセント |
| Warm Gold | `#BCAF92` | アイコン・補助ボタン |
| Dark Text | `#2C3E50` | 本文の濃いテキスト |
| Subtext Gray | `#6B7280` 系 | 補助テキスト |

### 3.3 配色ルール

- 背景は **クリームベージュ（#F9F5E7）と白の2層** を基調とし、セクションごとに交互配置する
- 主要見出しは **Primary Purple（#4B3E8E）**
- 4色（Green/Orange/Pink/Blue）は **同時に全て使う場面**（4つの柱の紹介、フィーチャーグリッド）で揃え、その他のセクションでは原則 1〜2色に抑える
- 赤（おたすけ Red）は **緊急・警告以外で広面積に使わない**
- グラデーションは「空」の比喩として用いる場合のみ（ヒーロー背景・CTA セクション等）

## 4. タイポグラフィ

### 4.1 ウェイトとサイズ

| 役割 | サイズ目安（Tailwind） | ウェイト |
|------|----------------------|---------|
| ヒーロータイトル | `text-3xl` 〜 `text-5xl` | Bold (700) |
| セクション見出し（H2） | `text-2xl md:text-3xl` | Bold (700) |
| カード見出し（H3） | `text-lg` 〜 `text-xl` | Bold (700) |
| 本文 | `text-sm` 〜 `text-base` | Normal (400) |
| キャプション | `text-xs` / `text-[10px]` | Semibold (600) |
| ボタンラベル | `text-sm` 〜 `text-base` | Bold (700) |

### 4.2 言語別の取り扱い

- 対応言語: **日本語（既定）／英語／中国語（簡体字）**
- 日本語見出しでは、長すぎる改行禁則を意識し、句読点や助詞で適切に改行できる長さに抑える
- 英語・中国語に展開する際は、日本語の比喩表現（例：「違いを、楽しもう。」）を **直訳ではなく意訳** する
- フォントはシステムフォントを基本とする（高速描画・OS最適化を優先）

## 5. ボイス＆メッセージ集

### 5.1 看板メッセージ（流用可）

| 場面 | 日本語 | 英語ニュアンス |
|------|--------|---------------|
| タグライン | 異文化の摩擦を、理解へ。 | From friction to understanding. |
| コンセプト | 違いを、楽しもう。 | Celebrate the differences. |
| 機能群 | 4つの「まな」で、文化をつなぐ | Four lenses to bridge cultures. |
| 個の尊重 | 目の前にいるのは、ただひとりの人間。 | Beyond cultures, there's a person. |
| 出会い | 文化は出会うための扉。 | Culture is a doorway to meet. |
| CTA | さあ、はじめよう | Let's begin. |

### 5.2 やってはいけない表現

- 「正しいマナー／間違ったマナー」と二項対立で言い切らない（「こうするとスムーズです」など柔らかく）
- 「外国人」を一括りに語らない（「あなた」「旅する人」「暮らす人」など個に向ける）
- 緊急機能を商業的に煽らない（おたすけは静かで信頼感のあるトーンを保つ）

## 6. 4つの柱（IA の中核）

| ID | 名称 | 意味 | 状態 | 機能色 |
|----|------|------|------|--------|
| shiru | しるまな | 知る — マナー・文化を学ぶ | 公開中 | Green `#85C441` |
| miru | みるまな | 見る — 文化コンテンツで楽しく学ぶ | 構想中 | Orange `#F26522` |
| kiku | きくまな | 聞く — 体験を共有するコミュニティ | 構想中 | Pink `#E4007F` |
| iku | いくまな | 行く — リアル文化体験イベント | 構想中 | Blue `#0099DD` |

補助柱:

| 名称 | 役割 | 機能色 |
|------|------|--------|
| おたすけ | 緊急ヘルプ（110/119、ステップガイド、相談窓口） | Red `#E60012` |
| まなアシスタント | AI 文化コンシェルジュ（Gemini） | Purple `#4B3E8E` |
| 防災マップ | 周辺の医療・避難施設等を Google Maps で表示 | （複合） |

**ナビゲーション順序**は原則 `しる → みる → きく → いく → おたすけ` を保つ。柱4つのカードは **2×2 グリッド（モバイル）** / **横並び（デスクトップ）** で揃える。

## 7. レイアウトと情報設計

### 7.1 全体構造（アプリ内）

```
[ヒーロー / 空 + ロゴ + タグライン]
   ↓
[コンセプト（理念の言葉）]
   ↓
[キーメッセージ 3点（個・違い・扉）]
   ↓
[4つの柱グリッド（しる・みる・きく・いく）]
   ↓
[アプリプレビュー / 機能スクリーンショット]
   ↓
[CTA（ブラウザで使う / インストール）]
```

### 7.2 ブレークポイントとグリッド

| 区分 | 想定幅 | 主な変化 |
|------|--------|---------|
| Mobile | 〜 767px | 4柱は 2×2、フッターは固定 4 アイコン、ドロワーは全画面 |
| Tablet | 768〜1023px | 2カラム化開始 |
| Desktop | 1024px〜 | 4柱は 1×4、サイドカラムに天気/カレンダー/マップを併置 |

最大コンテンツ幅の目安: 1200px。主要セクションは `max-w-4xl` 〜 `max-w-5xl` を中央寄せ。

### 7.3 余白とリズム

- セクション縦パディング: `py-14` 〜 `py-16`（モバイル）／`py-20`（デスクトップ）
- カードの内側パディング: `p-5` 〜 `p-6`
- カード間ギャップ: `gap-4`（モバイル）／`gap-6`（デスクトップ）

## 8. コンポーネント仕様

### 8.1 カード（4つの柱フィーチャーカード）

- 角丸: `rounded-2xl`
- 影: `shadow-lg`
- 背景: 機能色のフラット塗り（Green/Orange/Pink/Blue）
- テキスト: 白
- アイコンは上部中央、見出し（太字）、補足説明（`text-xs md:text-sm` / `opacity-90`）の縦並び
- hover で `scale-105`、active で `scale-95`

### 8.2 主要 CTA ボタン

| 種類 | 見た目 | 用途 |
|------|--------|------|
| Primary（インストール） | 背景 `#E4007F`、文字白、`rounded-2xl`、`shadow-lg` | 推奨アクション |
| Secondary（ブラウザで使う） | 背景白、文字 `#4B3E8E`、同形 | 代替アクション |
| Danger（おたすけ） | 背景 `#E60012`、文字白 | 緊急 |

- 高さ: `py-4`、横余白: `px-6`
- アイコン + テキスト（`gap-3`）

### 8.3 セクションヘッダ

- H2: `text-2xl md:text-3xl font-bold text-[#4B3E8E]`、中央寄せ
- サブ: `text-sm md:text-base text-gray-500 mb-10`

### 8.4 アイコン

- ライブラリ: `lucide-vue-next`
- サイズ: 一般は 20〜24px、フィーチャーカード内は 40〜48px
- ストロークウィジェット: デフォルト

### 8.5 モバイル ボトムナビ

4アイコン構成（DL / 設定 / プロフ / メニュー）。常時下部に固定。

### 8.6 ドロワー（サイドパネル）

- 幅: モバイル全幅、デスクトップ 360〜400px
- 角丸: 左側を `rounded-l-2xl`（PCで右から出る場合）
- アニメーション: 400ms `cubic-bezier(0.165, 0.84, 0.44, 1)`

## 9. モーション

| 種類 | 持続 | イージング | 用途 |
|------|------|-----------|------|
| ドロワー スライド | 400ms | `cubic-bezier(0.165, 0.84, 0.44, 1)` | サイドパネル |
| フェード | 400ms | ease | セクション現出 |
| ステップ切替 | 250ms | ease | ウィザード／オンボーディング |
| トースト | 300ms | ease | 通知 |
| ボタンプレス | 即時 | — | `active:scale-95` |
| 空アニメーション | 連続 | linear | 流れる雲と国旗 |

**ガイドライン:** モーションは「旅の体験」を演出する補助。過剰な視差効果や派手なローテーションは避ける。`prefers-reduced-motion` を尊重すること。

## 10. 画像とイラスト

| カテゴリ | 取り扱い |
|---------|---------|
| 写真 | 自然光、人物が含まれる場合は表情が温かいもの。記号的な「ニッポンらしさ（富士山・芸者）」のステレオタイプ多用は避ける |
| 国旗 | 多数の国を等価に扱う。特定国を中央や最大サイズで固定しない |
| イラスト | 線画＋淡い塗り。OK/NG 図解は ✓（緑）／✕（赤） のシンボルで明示 |
| スクリーンショット | スマホモックアップ（ノッチ表示あり）でアプリ画面を提示 |

## 11. アクセシビリティ

- コントラスト比 WCAG AA 以上（本文: 4.5:1、見出し大型: 3:1）
- 緊急機能（おたすけ）は色だけで意味を伝えず、テキスト・アイコンを併用
- フォーカスリングを消さない（PWA／キーボード操作対応）
- 言語属性は `<html lang>` に正しく設定（`ja` / `en` / `zh`）
- 多言語ルーティング: 既定 `ja` プレフィックスなし、`/en`、`/zh`

---

## 12. PWA／プラットフォーム表現

- ホーム画面アイコン: `/public/icons/logo.png` を基にしたマスカブル対応
- インストール CTA は「ブラウザで利用する」と並列に提示（強制しない）
- iOS / Android / PC それぞれの手動インストール手順を案内（ランディングのインストールモーダル参照）

## 13. 用途別チートシート

### 13.1 「ヒーロー」を作る時

- 背景: 空グラデーション（時間帯で変化）または `#F9F5E7`
- 中央: ロゴ（リング枠） + サービス名 + タグライン
- 右上: 言語切替

### 13.2 「機能を4つ並べる」時

- 必ず `しる → みる → きく → いく` の順
- 4色を全て使う
- 同一カード形状・同一アイコンサイズ・同一行数の説明文

### 13.3 「コール・トゥ・アクション」を置く時

- メイン（ピンク #E4007F、白文字）と サブ（白背景、Purple 文字）の2択を並べる
- メインの右側、または下に配置
- 文言は短く（「さあ、はじめよう」「ブラウザで利用する」など）

### 13.4 「緊急／注意」を表現する時

- 色: `#E60012`
- 言葉: 不安を煽らない。「困った時はここ」「ワンタップで連絡」など落ち着いた表現
- アイコン: ⚠️ や電話アイコンを併用

## 14. 派生コンテンツ生成時のチェックリスト

新しいビジュアル／コピー／画面を作る前に、以下を確認すること。

- [ ] 5つの理念（否定しない／争わない／個を尊重／共生／楽しく学ぶ）に反していないか
- [ ] 4つの柱と機能色のマッピングが正しいか
- [ ] 既定の配色ルール（ベージュ＋白の2層、Purpleの見出し）を踏襲しているか
- [ ] タグラインや看板メッセージを **そのまま流用** しているか（独自に書き換えていないか）
- [ ] ステレオタイプ表現を使っていないか
- [ ] 多言語展開時に直訳に陥っていないか
- [ ] アクセシビリティ（コントラスト・色だけに依存しない情報）を満たしているか
