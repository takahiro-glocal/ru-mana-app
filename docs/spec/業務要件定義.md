## 1. 業務概要

### 1.1 業務の目的

異文化理解プラットフォームとして、以下の業務を支援する：

1. **文化学習支援** — 訪日外国人が日本の文化・マナーを楽しく学べる場を提供
2. **コミュニティ運営** — ユーザー同士が体験を共有し、相互理解を深めるコミュニティを運営
3. **緊急時支援** — 旅行中のトラブル発生時に迅速な対応ガイドを提供
4. **AI文化ガイド** — AIアシスタントによるリアルタイムの文化相談対応

### 1.2 業務フロー全体像

```
[訪日外国人] → アプリ訪問 → ブラウザ言語自動検出 → 言語リダイレクト
                                    │
                         ┌──────────┼──────────────────────┐
                         ▼          ▼                      ▼
                    しるまな     おたすけ           まなアシスタント
                    (学習)      (緊急ヘルプ)         (AI相談)
                         │
              ┌──────────┼──────────┐
              ▼          ▼          ▼
         カテゴリ閲覧  スレッド投稿  ガイド閲覧
              │          │
              ▼          ▼
         AI翻訳表示   コメント・いいね
```

## 2. 業務フロー詳細

### 2.1 ユーザー登録・認証フロー

```
[未認証ユーザー]
  ├─ 閲覧のみ可能（スレッド・ガイド・おたすけ）
  │
  ├─ 投稿・コメント・いいね → ログインモーダル表示
  │     ├─ Google OAuth（ポップアップ → フォールバック: リダイレクト）
  │     └─ メール&パスワード（新規登録 or ログイン）
  │
  └─ 認証成功
        ├─ Firebase Auth にセッション保持（ブラウザローカル永続化）
        ├─ DiceBear API で自動アバター生成（photoURL未設定の場合）
        └─ Firestore にユーザードキュメント作成（初回のみ）
```

### 2.2 しるまな業務フロー

#### 2.2.1 コンテンツ閲覧

```
[ユーザー]
  → ホーム画面「しるまな」タップ
  → カテゴリ一覧表示（8カテゴリ）
  → カテゴリ選択
  → タブ切替
    ├─ Q&A（スレッド一覧）
    │    → スレッド選択 → 投稿一覧表示（リアルタイム更新）
    │    → 非日本語ユーザーの場合: タイトル自動翻訳（3層キャッシュ）
    │
    └─ るうまなガイド
         → マナーガイド一覧（OK例/NG例付き）
         → i18n による多言語表示
```

#### 2.2.2 投稿フロー

```
[認証済みユーザー]
  → スレッド作成
    ├─ タイトル・本文を入力
    ├─ Firestore threads コレクションに保存
    └─ 初回投稿（Post）を同時作成

  → コメント投稿
    ├─ 本文を入力（返信引用を含む場合あり）
    ├─ posts サブコレクションに保存
    └─ スレッドの postCount + updatedAt を更新

  → いいね
    ├─ likes カウントをインクリメント（Firestore）
    └─ ユーザーごとの重複チェック（localStorage）

  → 編集・削除（投稿者本人のみ）
```

### 2.3 AI翻訳業務フロー

```
[ユーザー操作: 翻訳ボタンクリック]
  │
  ├─ 1. メモリキャッシュ確認 → ヒット → 即時表示
  │
  ├─ 2. Firestore キャッシュ確認 → ヒット → メモリに保存 → 表示
  │
  └─ 3. Gemini API 呼び出し（クライアントサイド）
       → 翻訳結果を取得
       → メモリ + Firestore に保存
       → 表示

  ※ キャッシュキー: テキスト+言語のハッシュ値 (tr_{hashCode})
```

### 2.4 AIチャット業務フロー

```
[ユーザー: チャットパネル開く]
  → テキスト入力 → 送信
  │
  ├─ システムプロンプト構築（ナレッジベース + 理念 + 言語指示）
  ├─ 会話履歴（直近20件）を添付
  ├─ Gemini API (gemini-2.5-flash) にリクエスト
  └─ 応答をメッセージリストに追加・表示

  ※ るうまなの理念（否定しない・争わない・個を尊重）がシステムプロンプトに組み込まれている
```

### 2.5 おたすけ業務フロー

```
[ユーザー: 緊急事態発生]
  → ホーム画面「In Trouble」ボタン
  → おたすけ画面
    ├─ 緊急電話（110/119）ワンタップ発信
    │    → 確認ダイアログ → tel: スキームで発信
    │
    ├─ カテゴリ選択（病気・紛失・災害）
    │    → ステップバイステップガイド表示（モーダル）
    │
    ├─ 現在地コピー
    │    → クリップボードにコピー → 通報時に利用
    │
    └─ 相談窓口
         ├─ ひまわり（03-5320-7744）24時間多言語対応
         └─ Visitor Hotline（0570-000-911）
```

### 2.6 防災マップ業務フロー

```
[ユーザー]
  → 防災マップ画面
  → Google Maps 表示（現在地ベース）
  → カテゴリフィルタ（医療/避難/食料品/警察消防/公共施設）
  → マーカータップ → 施設詳細表示
  → 現在地追跡ボタン → マップ中心を現在地に移動
```

## 3. ユーザーロール定義

| ロール | 権限 |
|--------|------|
| 未認証ユーザー（ゲスト） | 閲覧のみ（スレッド・ガイド・おたすけ・防災マップ・チャット） |
| 認証済みユーザー | 上記 + スレッド作成・投稿・コメント・いいね・プロフィール編集・ポイント |
| 公式スタッフ | 上記 + isOfficial バッジ付き投稿 |

---

## 4. 業務ルール

### 4.1 投稿ルール

| ルール | 内容 |
|--------|------|
| 投稿の編集・削除 | 投稿者本人のみ可能 |
| いいねの重複 | localStorage で1ユーザー1回に制限 |
| スレッド表示順 | 更新日時の降順 |
| ホームの最新スレッド | 全カテゴリから最新5件を表示 |
| 閲覧履歴の上限 | 最大50件（最新順、重複は最上部に移動） |
| 会話履歴の上限 | AIチャットは直近20メッセージを保持 |

### 4.2 翻訳ルール

| ルール | 内容 |
|--------|------|
| 翻訳対象 | スレッドタイトル、投稿本文 |
| 自動翻訳 | デフォルト言語以外のユーザーにはタイトルを自動翻訳 |
| 手動翻訳 | 投稿本文は「翻訳を表示」ボタンで個別翻訳 |
| キャッシュ永続化 | Firestore の translations コレクションに保存 |

### 4.3 ポイントルール

| ルール | 内容 |
|--------|------|
| ポイント付与 | 活動（投稿・コメント等）に応じて付与 |
| ポイント履歴 | 最新20件を表示 |
| ポイントの用途 | 現時点では表示のみ（将来の拡張用） |

ホーム (/)
  ├─ しるまな (/shiru)
  │    └─ カテゴリ (/shiru/category/{cid})

  ├─ みるまな (/miru) → Coming Soon リダイレクト

  └─ ドロワー（サイドパネル）

## 6. 外部連携

| 連携先 | 用途 | 方式 |
|--------|------|------|
| Firebase Authentication | ユーザー認証 | クライアントSDK |
| Cloud Firestore | データベース | クライアントSDK（リアルタイムリスナー） |
| Firebase Storage | アバター画像保存 | クライアントSDK |
| Google Generative AI (Gemini) | AIチャット・翻訳 | クライアントSDK（@google/generative-ai） |
| Google Maps API | 地図表示・位置情報 | JavaScript SDK |
| OpenWeatherMap API | 天気情報取得 | REST API |
| DiceBear API | 自動アバター生成 | CDN URL |
