## 1. データベース概要

Cloud Firestore を使用したドキュメント指向データベース。リアルタイムリスナーによるライブデータ同期と、ローカルキャッシュ（`persistentLocalCache`）によるオフライン対応を備える。

---

## 2. コレクション定義

### 2.1 threads（スレッド）

```
Firestore パス: /threads/{threadId}
```

| フィールド | 型 | 必須 | 説明 |
|-----------|------|------|------|
| categoryId | `string` | ○ | カテゴリID（transport, public, spa, cafe, shopping, hotel, culture, trash, new） |
| title | `string` | ○ | スレッドタイトル |
| createdAt | `Timestamp` | ○ | 作成日時（serverTimestamp） |
| updatedAt | `Timestamp` | ○ | 更新日時（serverTimestamp、投稿追加で更新） |
| postCount | `number` | ○ | 投稿数（increment で管理） |

**インデックス:**

| フィールド | 順序 | 用途 |
|-----------|------|------|
| categoryId (ASC) + updatedAt (DESC) | 複合 | カテゴリ別スレッド一覧 |
| updatedAt (DESC) | 単一 | 最新スレッドフィード |

---

### 2.2 threads/{threadId}/posts（投稿）

```
Firestore パス: /threads/{threadId}/posts/{postId}
```

| フィールド | 型 | 必須 | 説明 |
|-----------|------|------|------|
| body | `string` | ○ | 投稿本文 |
| userId | `string` | ○ | 投稿者のFirebase UID |
| userName | `string` | ○ | 投稿者の表示名 |
| userIcon | `string` | ○ | 投稿者のアバターURL |
| createdAt | `Timestamp` | ○ | 作成日時（serverTimestamp） |
| likes | `number` | ○ | いいね数（デフォルト: 0） |
| replyToId | `string` | - | 返信先の投稿ID |
| replyToName | `string` | - | 返信先の投稿者名 |
| replyToText | `string` | - | 返信先の投稿本文（引用表示用） |
| isOfficial | `boolean` | - | 公式スタッフ投稿フラグ |

**インデックス:**

| フィールド | 順序 | 用途 |
|-----------|------|------|
| createdAt (ASC) | 単一 | 投稿の時系列表示 |

**サブコレクション collectionGroup クエリ:**

- `getUserPosts(userId)`: collectionGroup('posts').where('userId', '==', userId) で全スレッドの投稿を横断検索

---

### 2.3 users（ユーザー）

```
Firestore パス: /users/{userId}
```

| フィールド | 型 | 必須 | 説明 |
|-----------|------|------|------|
| points | `number` | ○ | 現在のポイント残高 |
| displayName | `string` | - | 表示名（Firebase Authと同期） |
| photoURL | `string` | - | アバターURL（Firebase Authと同期） |

---

### 2.4 users/{userId}/point_history（ポイント履歴）

```
Firestore パス: /users/{userId}/point_history/{historyId}
```

| フィールド | 型 | 必須 | 説明 |
|-----------|------|------|------|
| amount | `number` | ○ | ポイント変動量（正: 付与、負: 消費） |
| reason | `string` | ○ | 変動理由 |
| createdAt | `Timestamp` | ○ | 発生日時（serverTimestamp） |

**インデックス:**

| フィールド | 順序 | 用途 |
|-----------|------|------|
| createdAt (DESC) | 単一 | 最新順履歴表示（limit 20） |

---

### 2.5 translations（翻訳キャッシュ）

```
Firestore パス: /translations/{cacheId}
```

| フィールド | 型 | 必須 | 説明 |
|-----------|------|------|------|
| sourceText | `string` | ○ | 翻訳元テキスト |
| targetLang | `string` | ○ | 翻訳先言語コード（en, zh） |
| translatedText | `string` | ○ | 翻訳結果テキスト |
| createdAt | `Date` | ○ | キャッシュ作成日時 |

**ドキュメントIDの生成ルール:**

```
cacheId = tr_{Math.abs(hashCode(sourceText + '::' + targetLang)).toString(36)}
```

ハッシュ関数: Java風の文字列ハッシュ（`(hash << 5) - hash + charCode`）

---

### 2.6 feedback（フィードバック）

```
Firestore パス: /feedback/{feedbackId}
```

| フィールド | 型 | 必須 | 説明 |
|-----------|------|------|------|
| content | `string` | ○ | フィードバック内容 |
| userId | `string` | ○ | 送信者のFirebase UID |
| userName | `string` | ○ | 送信者の表示名 |
| createdAt | `Timestamp` | ○ | 送信日時（serverTimestamp） |
| status | `string` | ○ | ステータス（`new` / `reviewed`） |

## 3. ER図（概念）

```
[threads] ─────< [posts]           (1:N サブコレクション)
    │
    │ categoryId で論理的にグループ化
    │
[users] ─────< [point_history]     (1:N サブコレクション)
    │
    │ userId で posts と論理的に関連
    │
[translations]                      (独立コレクション)
[feedback]                          (独立コレクション)
```

## 6. セキュリティルール（推奨）

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // threads: 誰でも読み取り可、認証済みユーザーのみ書き込み
    match /threads/{threadId} {
      allow read: if true;
      allow create: if request.auth != null;
      allow update: if request.auth != null;

      // posts: 誰でも読み取り可、認証済みのみ作成、本人のみ編集/削除
      match /posts/{postId} {
        allow read: if true;
        allow create: if request.auth != null;
        allow update, delete: if request.auth.uid == resource.data.userId;
      }
    }

    // users: 本人のみ読み書き
    match /users/{userId} {
      allow read, write: if request.auth.uid == userId;

      match /point_history/{historyId} {
        allow read: if request.auth.uid == userId;
        allow create: if request.auth.uid == userId;
      }
    }

    // translations: 誰でも読み書き（キャッシュ用）
    match /translations/{docId} {
      allow read, write: if true;
    }

    // feedback: 認証済みのみ作成、読み取り不可
    match /feedback/{docId} {
      allow create: if request.auth != null;
      allow read: if false;
    }
  }
}
```
