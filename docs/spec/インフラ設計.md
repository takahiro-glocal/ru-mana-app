## 1. インフラ構成概要

### 1.1 全体構成図

```
[ユーザーブラウザ]
      │ HTTPS
      ▼
┌─────────────────────────┐
│ Firebase Hosting (CDN)  │  ← 静的ファイル配信
│ *.web.app / カスタムドメイン │
└───────────┬─────────────┘
            │ SPA (index.html)
            ▼
┌──────────────────────────────────────────────┐
│          クライアントサイド (ブラウザ)         │
│                                              │
│  ┌──────────┐  ┌──────────┐  ┌────────────┐ │
│  │Firebase   │  │Gemini API│  │Google Maps │ │
│  │Auth/FS/ST│  │(直接呼出)│  │API         │ │
│  └─────┬────┘  └─────┬────┘  └─────┬──────┘ │
└────────┼─────────────┼─────────────┼────────┘
         │             │             │
         ▼             ▼             ▼
┌──────────────┐ ┌───────────┐ ┌───────────────┐
│Firebase      │ │Google AI  │ │Google Maps    │
│Backend       │ │Platform   │ │Platform       │
│(Google Cloud)│ │           │ │               │
├──────────────┤ └───────────┘ └───────────────┘
│Authentication│
│Firestore     │
│Storage       │
│Analytics     │
└──────────────┘
```

### 1.2 アーキテクチャ特徴

| 特徴 | 説明 |
|------|------|
| サーバーレス | 自前サーバー不要。全てマネージドサービス |
| SPA | Nuxt 3 Static Preset で完全な静的ファイル生成 |
| CDN配信 | Firebase Hosting のグローバルCDN |
| クライアントサイドAPI | AI/DB/認証は全てクライアントSDKから直接アクセス |

## 2. Firebase プロジェクト構成

### 2.1 使用サービス

| サービス | 用途 | プラン |
|----------|------|--------|
| Firebase Hosting | 静的ファイルホスティング | Spark（無料）/ Blaze（従量課金） |
| Firebase Authentication | ユーザー認証 | 無料（月間MAU制限あり） |
| Cloud Firestore | NoSQLデータベース | Spark / Blaze |
| Firebase Storage | ファイルストレージ（アバター） | Spark / Blaze |
| Firebase Analytics | アクセス解析 | 無料 |

### 2.2 Firebase Hosting 設定

```json
{
  "hosting": {
    "public": ".output/public",
    "ignore": ["firebase.json", "**/.*", "**/node_modules/**"],
    "rewrites": [
      {
        "source": "**",
        "destination": "/index.html"
      }
    ]
  }
}
```

- **public**: Nuxt Static ビルドの出力ディレクトリ
- **rewrites**: SPAフォールバック（全パスを `index.html` に）
- **Cloud Functions**: 使用しない（組織IAMポリシー制約のため）

### 2.3 Firestore 設定

| 項目 | 値 |
|------|------|
| ロケーション | asia-northeast1（東京） |
| ローカルキャッシュ | `persistentLocalCache` 有効 |
| タブ管理 | `persistentSingleTabManager` |
| リアルタイム | `onSnapshot` リスナー使用 |

### 2.4 Firebase Storage 設定

| 項目 | 値 |
|------|------|
| バケット | デフォルト（`{projectId}.appspot.com`） |
| 用途 | ユーザーアバター画像 |
| パス | `avatars/{uid}` |
| 最大サイズ | 2MB（クライアントサイドバリデーション） |

## 4. セキュリティ構成

### 4.1 通信セキュリティ

```
[ブラウザ] ─── HTTPS ──→ [Firebase Hosting]
[ブラウザ] ─── HTTPS ──→ [Firestore]
[ブラウザ] ─── HTTPS ──→ [Gemini API]
[ブラウザ] ─── HTTPS ──→ [Google Maps API]
[ブラウザ] ─── HTTPS ──→ [OpenWeatherMap API]
```

全通信はHTTPS。Firebase Hosting は自動的にHTTPS証明書を管理。

### 4.2 APIキー保護戦略

| キー | 保護方式 | 設定場所 |
|------|----------|----------|
| Firebase API Key | Firebase Security Rules | Firebase Console |
| Gemini API Key | HTTPリファラー制限 | Google Cloud Console > API & Services > Credentials |
| Google Maps API Key | HTTPリファラー制限 + API制限 | Google Cloud Console |
| OpenWeatherMap API Key | ドメイン制限 | OpenWeatherMap ダッシュボード |

### 4.3 HTTPリファラー制限（推奨設定）

```
許可するリファラー:
  - ru-mana-web.web.app/*
  - localhost:3000/*  （開発用）
  - {カスタムドメイン}/*
```

### 4.4 Firebase Security Rules

Firestore Security Rules でコレクション単位のアクセス制御を実施（詳細はデータベース定義書を参照）。

## 5. デプロイフロー

### 5.1 手動デプロイ

```bash
# 1. 環境変数の設定
cp env.dev .env
# .env に本番値を設定

# 2. ビルド＆デプロイ
npm run deploy
# 内部: nuxt build && firebase deploy --only hosting
```

### 5.2 デプロイ出力

```
ビルド: .output/public/ (静的HTML/JS/CSS)
デプロイ先: Firebase Hosting
URL: https://{project-id}.web.app
```

### 5.3 デプロイ前チェックリスト

| 項目 | 確認内容 |
|------|----------|
| `.env` | 全環境変数が本番値で設定されていること |
| APIキー制限 | HTTPリファラー制限が本番ドメインに設定済み |
| Firestoreルール | セキュリティルールがデプロイ済み |
| Storageルール | アバターアップロードのルールが設定済み |
| ビルド | `npm run build` がエラーなく完了すること |

## 7. バックアップ・復旧

| 対象 | 方式 | 頻度 |
|------|------|------|
| Firestoreデータ | Firebase自動バックアップ | 日次（設定が必要） |

```bash

# 直前のデプロイにロールバック

```

## 8. ネットワーク構成

```
[ユーザー]
    │
    ▼ DNS
[Firebase Hosting CDN]
    │ エッジサーバー（グローバル分散）
    │
    ├── 静的ファイル配信（HTML/JS/CSS/画像）
    │
    └── SPA内からの外部API呼び出し:
        ├── Firestore: firestore.googleapis.com (TCP 443)
        ├── Auth: identitytoolkit.googleapis.com (TCP 443)
        ├── Storage: firebasestorage.googleapis.com (TCP 443)
        ├── Gemini: generativelanguage.googleapis.com (TCP 443)
        ├── Maps: maps.googleapis.com (TCP 443)
        └── Weather: api.openweathermap.org (TCP 443)
```

全ての通信はTCP 443（HTTPS）のみ。特別なポート開放は不要。
